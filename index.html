O mnie

Od 15 lat działam w zakresie IT.
Na początku były to proste strony HTML i CSS pisane w notatniku. Później, będąc na studiach, poznałem języki programowania takie jak C, C++, Java, Python. Po ukończeniu studiów pracowałem w Sulimo Software jako fullstack, gdzie rozwijałem projekt w PHP na frameworku YII, a następnie przeszedłem na stanowisko tworzenia interaktywnych dokumentów. W międzyczasie podjąłem pracę na uczelni, gdzie prowadziłem wiele różnych zajęć z dziedziny informatyki i matematyki. Ponadto pisałem aplikacje w Android/Kotlin dla własnego użytku oraz zrobiłem kurs React Native. Dzięki temu mogłem podjąć pracę jako programista aplikacji mobilnych z wykorzystaniem RN, gdzie rozwijałem aplikację, a napotykane problemy skuteczne rozwiązywałem.
Prywatnie aktualnie tworzę aplikację, która będzie bazą do stworzenia wszystkich pomysłów moich i moich znajomych.

Projekty autorskie

Aplikacja mobilna

Aplikacja mobilna napisana w React Native z wykorzystaniem TypeScript. Aplikacja posiada połączenie z API, zarządzanie globalnym stanem, listy, karuzele, filtrację, zagnieżdżoną nawigację, formularze z walidacją. Aplikacja nie ma aktualnie przeznaczenia, będzie szablonem do napisania autorskich pomysłów.

CMS

CMS napisany w React z wykorzystaniem TypeScript. Aplikacja posiada połączenie z API, zarządzanie globalnym stanem oraz pełne zarządzanie CRUD. Aplikacj zrobiona jest tak, aby nowe powiązania z API były banalnie proste do wygenerowania. Lista jest paginowana, a wyświetlane pola zależą od przekazanej tablicy. Komponent obsługi CRUD jest zrobiony generycznie, dlatego wystarczy że dane z API będą miały analogiczny schemat. W na drugim screenie pokazane jest użycie komponentu szablonowego - wystarczy przekazać ścieżkę do API, lokalną ścieżkę oraz kolumny które chce się wyświetlić.

API

API napisane w NestJS z wykorzystaniem TypeScript. Aplikacja posiada nierelacyjną bazę danych MongoDB. Crud generowany jest za pomocą jednego polecenia, dostosowanie jest szybkie i mało skomplikowane. Wszystkie endpointy uwzględnione są w automatycznie generowanej dokumentacji.
