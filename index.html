O mnie

Od 15 lat działam w dziedzinie IT, na początku były to proste strony napisane w html i css w notatniku. Później na studiach poznałem języki programowania takie jak C, C++, Java, Python. Po studiach znalazłem zatrudnienie w Sulimo Software, było to stanowisko fullstack, gdzie rozwijałem projekt w PHP na frameworku YII. Później w tej samej firmie przeszedłem na stanowisko tworzenia interaktywnych dokumentów. W międzyczasie zatrudniłem się na uczelni gdzie prowadziłem wiele różnych zajęć z dziedziny informatyki i matematyki. Prywatnie pisałem aplikacje w android/kotlin dla własnego użytku. Następnie zrobiłem kurs react native i podjąłem pracę jako programista aplikacji mobilnych w react native i przez niecałe 2 lata rozwijałem aplikację napotykając się na wiele problemów i skutecznie je rozwiązując. Aktualnie prywatnie tworzę aplikację, która będzie bazą do stworzenia wszystkich pomysłów moich i moich znajomych.

Projekty autorskie

Aplikacja mobilna

Aplikacja mobilna napisana w React Native z wykorzystaniem typescript. Aplikacja posiada połączenie z API, zarządzanie globalnym stanem, listy, karuzele, filtrację, zagnieżdżoną nawigację, formularze z walidacją. Aplikacja nie ma aktualnie przeznaczenie, będzie szablonem do napisania autorskich pomysłów.

CMS

CMS napisany w React z wykorzystaniem typescript. Aplikacja posiada połączenie z API, zarządzanie globalnym stanem oraz pełne zarządzanie CRUD. Aplikacj zrobiona jest tak aby nowe powiązania z API były banalnie proste do wygenerowania. Lista jest paginowana, a wyświetlane pola zależą od przekazanej tablicy. Komponent obsługi CRUD jest zrobiony generycznie, dlatego wystarczy że dane z api będą miały analogiczny schemat. W na drugim screenie pokazane jest użycie komponentu szablonowego, wystarczy przekazać ścieżkę do api, lokalną ścieżkę oraz kolumny które chce się wyświetlić.

API

API napisane w NestJS z wykorzystaniem typescript. Aplikacja posiada nierelacyjną bazę danych MongoDB. Crud generowany jest za pomocą jednego polecenia, dostosowanie jest szybkie i mało skomplikowane. Wszystkie endpointy uwzględnione są w automatycznie generowanej dokumentacji.
